Supported credentials depending on service type

Amazon

  • Username — AWS access key ID
  • Token — AWS secret access key

* make sure your AWS user have proper permission policy attached, e.g. "AmazonEC2FullAccess"

Supported Amazon options are: "region" (default: "us-east-1")

Azure

  • Username — Azure subscription id in the form of GUID
  • Certificate — Azure X509 certificate file in .pem form

Supported Azure options are: "location" (default: "Central US"), "cloud_service_name" (optional, if defined all connected SPLs will operate in the defined cloud service group), "images_regex" (optional, an regex to limit images list)

DigitalOcean

  • Token — DigitalOcean personal access token

Jira

  • Backend url — Jira host (e.g. http://example.atlassian.net/)
  • Username
  • Password

GitLab

  • Backend url — GitLab host (e.g. https://git.example.com/)
  • Username — Username or Email
  • Password
  • Token — Private token (will be used instead of username/password if supplied)

OpenStack

  • Backend url — Keystone auth URL (e.g. http://keystone.example.com:5000/v2.0)
  • Username
  • Password

Supported OpenStack options are: "tenant_name" (default: "admin"), "availability_zone", "cpu_overcommit_ratio" (default: 1) and "external_network_id" (ID of OpenStack external network that will be connected to new service tenants)

Oracle

  • Backend url — Oracle EM host (e.g. https://oracle.example.com:7802/em)
  • Username
  • Password

SugarCRM

  • Backend url - URL of OpenStack service project link that will be used for sugarCRM resources creation (required, e.g.: http://example.com/api/openstack-service-project-link/1/)
  • Username - NodeConductor user username (e.g. Username)
  • Password NodeConductor user password (e.g. Password)

    Supported SugarCRM options are:
  • "security_groups_names" - List of CRMs OpenStack instance security groups names (default: ["http"])
  • "image_name" - CRMs OpenStack instance image name (default: "sugarcrm")
  • "min_cores" - Minimum amount of cores for CRMs OpenStack instance (default: 2)
  • "min_ram" - Minimum amount of ram for CRMs OpenStack instance (default: 2048 MB)
  • "system_size" - Storage volume size CRMs OpenStack instance (default: 32768 MB)
  • "user_data" - User data that will be passed to CRMs OpenStack instance on creation, word {password} will be replaced with auto-generated admin password (default: "#cloud-config:\nruncmd:\n - [bootstrap, -p, {password}])

SaltStack

  • Backend url — URL for salt master API (required, e.g.: http://salt-master.example.com:8080)
  • Username
  • Password

Zabbix

  • Backend url - Zabbix API URL (e.g. http://example.com/zabbix/api_jsonrpc.php)
  • Username - Zabbix user username (e.g. admin)
  • Password - Zabbix user password (e.g. zabbix)

    Supported SugarCRM options are:
  • "host_group_name" - Zabbix host group name for registered hosts. (default: "nodeconductor")
  • "interface_parameters" - Parameters for hosts interface. (default: {"dns": "", "ip": "0.0.0.0", "main": 1, "port": "10050", "type": 1, "useip": 1})
  • "templates_names" - List of Zabbix hosts templates. (default: ["nodeconductor"])
